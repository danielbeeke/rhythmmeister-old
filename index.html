Introduction

Print has solved typography problems ages ago. Things like the baseline and kerning are industry standards, the web however evolving from hypertext not so much.

The main problem lies in the use of line-height instead of a base-line. RhythmMeister provides a sass mixin so you can use the base-line instead of the line-height.

It provides this via a simple trick. It sets the line-height to the selected rows and than it adds a little padding to the top and the bottom.

Installation

You can install RhythmMeister via bower.

Just use bower install rhythmmeister --save.
Afterwards you can include the _rhythmmeister.scss.

However bower is not needed to use RhythmMeister, it would be good to check it out, but you can always just download the stylesheets from github.

Initialisation

For RhythmMeister to work you need to set to variables. The first is $document-base-line, this one tells what the height of your vertical grid is. The second variable is a map of all your fonts and their properties.

It will be a little bit difficult to find the baseline, because the baseline will be the percentage from the font object from the top of the object till the baseline. This will be different each time you change the font size or the rows.

With the rows you can set how much rows it should take to render the font.

$document-base-line: 24px;

$font-variants: (
  m: (
    font-size: 26px,
    base-line: 0.71,
    rows: 2
  ),
  l: (
    font-size: 32px,
    base-line: 0.76,
    rows: 2
  )
);

vertical-rhythm()

This is the main function of RhythmMeister, you need to use this for setting padding and margins on font objects (h1, h2, span, p, etc). The reason all these inputs need to be in one function is because we use the padding for our trick and maybe you still want to use it for adding extra rows than we need to add the rows to the small padding.

The first for four parameters are unitless. You just need to supply a number. The font variant needs to be a key out of the $font-variants variable. The last two are fixes to substract the border of the margin. You just need to supply the pixels to it.

 @include vertical-rhythm(
  $padding-top: 0,
  $padding-bottom: 0,
  $margin-top: 0,
  $margin-bottom: 0,
  $font-variant: '',
  $border-top-width: 0,
  $border-bottom-width: 0
);
